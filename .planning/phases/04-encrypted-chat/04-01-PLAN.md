Create src/hooks/useChat.ts following the research Pattern 2 and Pattern 3:

```typescript
export function useChat(
  peerRef: RefObject<Peer | null>,
  roomId: string
): { sendMessage: (text: string) => Promise<void>; isReady: boolean }
```

Implementation details:
- Use refs for: connRef (DataConnection), keyPairRef (CryptoKeyPair), sharedKeyRef (CryptoKey), pendingMessages (string[])
- Track `isReady` with useState (true when sharedKeyRef.current is set)
- `setupConnection(conn)` callback registers all listeners synchronously:
  - `conn.on('open', async () => { ... })` — generate keypair, export public key, send key-exchange message. After registering, check `if (conn.open)` and manually trigger the open logic as fallback (Pitfall 1 from research).
  - `conn.on('data', async (raw) => { ... })` — handle key-exchange (import key, derive shared, flush pending, setIsReady(true)) and message (decrypt, addMessage) types
  - `conn.on('close', () => { ... })` — null refs, setIsReady(false)
  - `conn.on('error', (err) => console.error('DataConnection error:', err))`

- useEffect for DataConnection setup:
  - Gate on peerRef.current, peer.open, and peerId from store (same pattern as useCall)
  - Determine isCreator from peer.id === roomId
  - Joiner: `peer.connect(roomId, { serialization: 'json', reliable: true })` then `setupConnection(conn)`
  - Creator: `peer.on('connection', (conn) => setupConnection(conn))`
  - Cleanup: close connRef.current, remove peer 'connection' listener

- `sendMessage` async function:
  - Guard: if !sharedKeyRef.current or !connRef.current, return early
  - Encrypt text with encryptMessage(sharedKeyRef.current, text)
  - conn.send({ type: 'message', payload: encrypted })
  - addMessage({ from: 'local', text, timestamp: Date.now() })
