    - Test: startScreenShare calls getDisplayMedia and replaces the video sender track
    - Test: startScreenShare attaches onended handler to display track
    - Test: stopScreenShare stops the display track and restores the camera track via replaceTrack
    - Test: when displayTrack.onended fires, camera track is restored automatically
    - Test: isScreenSharing state toggles correctly on start/stop
  </behavior>
  <action>
1. **Expose `callRef` from `useCall`:** Modify `src/hooks/useCall.ts` to include `callRef` in the return object: `return { callRef, remoteStreamRef, hangUp }`. This is a minimal one-line change — add `callRef` to the return object. Update the return type comment if any.

2. Create `src/hooks/useScreenShare.ts`:
   - Accept `callRef: RefObject<MediaConnection | null>` and `streamRef: RefObject<MediaStream | null>` as parameters
   - `isScreenSharing` state via `useState(false)`
   - `displayTrackRef = useRef<MediaStreamTrack | null>(null)` — store display track in ref, NOT state (anti-pattern from RESEARCH.md)
   - `startScreenShare`: async function that:
     a. Calls `navigator.mediaDevices.getDisplayMedia({ video: true, audio: false })`
     b. Gets the video track from the display stream
     c. Accesses `callRef.current?.peerConnection`
     d. Finds the video sender via `pc.getSenders().find(s => s.track?.kind === 'video')`
     e. Calls `sender.replaceTrack(displayTrack)` — NO renegotiation (critical: do NOT call `peer.call()` again)
     f. Sets `displayTrackRef.current = displayTrack`
     g. Sets `isScreenSharing(true)`
     h. Attaches `displayTrack.onended = () => stopScreenShare()` for OS "Stop Sharing" button
     i. Wraps in try/catch — if user cancels the picker, catch the error silently (DOMException with name 'NotAllowedError')
   - `stopScreenShare`: async function that:
     a. Calls `displayTrackRef.current?.stop()` to release the display capture
     b. Sets `displayTrackRef.current = null`
     c. Gets camera track from `streamRef.current?.getVideoTracks()[0]`
     d. Replaces back via `sender.replaceTrack(cameraTrack)`
     e. Sets `isScreenSharing(false)`
   - Use `useCallback` for both functions with correct deps
   - Return `{ isScreenSharing, startScreenShare, stopScreenShare }`

3. Create `src/hooks/useScreenShare.test.ts`:
   - Mock `navigator.mediaDevices.getDisplayMedia` to return a fake stream with a video track
   - Mock `peerConnection.getSenders()` to return a sender with `replaceTrack` spy
   - Create fake `callRef` and `streamRef` refs
   - Test all behaviors listed above
   - For the `onended` test: capture the handler assigned to `displayTrack.onended`, then invoke it, and verify `replaceTrack` was called with the camera track
  </action>

    - Test: useNetworkQuality returns 'good' when packet loss < 2% and RTT < 150ms
    - Test: useNetworkQuality returns 'fair' when packet loss < 8% and RTT < 400ms
    - Test: useNetworkQuality returns 'poor' when packet loss >= 8% or RTT >= 400ms
    - Test: NetworkQualityBadge renders "Good" with green styling for quality='good'
    - Test: NetworkQualityBadge renders "Fair" with yellow styling for quality='fair'
    - Test: NetworkQualityBadge renders "Poor" with red styling for quality='poor'
    - Test: NetworkQualityBadge renders nothing when quality=null
  </behavior>
  <action>
1. Create `src/hooks/useNetworkQuality.ts`:
   - Export type `NetworkQuality = 'good' | 'fair' | 'poor' | null`
   - Accept `callRef: RefObject<MediaConnection | null>` and `active: boolean`
   - `quality` state via `useState<NetworkQuality>(null)`
   - `prevStatsRef = useRef<Map<string, any>>(new Map())` for storing previous stats report
   - `useEffect` that runs when `active` is true:
     a. Set up `setInterval` at 2000ms
     b. Get `pc = callRef.current?.peerConnection`, bail if null
     c. Call `await pc.getStats()`
     d. Iterate the report:
        - For `candidate-pair` with `state === 'succeeded'`: read `currentRoundTripTime` as RTT
        - For `outbound-rtp` with `kind === 'video'`: compute delta `nackCount` and `packetsSent` from previous stats. Packet loss = deltaNack / deltaSent (null-guard all fields)
     e. Store current stats in prevStatsRef
     f. Classify: loss < 0.02 && rtt < 0.15 → 'good', loss < 0.08 && rtt < 0.4 → 'fair', else → 'poor'
     g. Set quality state
   - Return `quality`
   - Cleanup: clearInterval on unmount or when active becomes false. Reset quality to null and clear prevStatsRef.

2. Create `src/components/NetworkQualityBadge.tsx`:
   - Accept `{ quality: NetworkQuality }` prop
   - If quality is null, return null (render nothing)
   - Use the `Badge` component from `@/components/ui/badge`
   - Map quality to display: `{ good: { label: 'Good', className: 'bg-green-500/20 text-green-400 border-green-500/30' }, fair: { label: 'Fair', className: 'bg-yellow-500/20 text-yellow-400 border-yellow-500/30' }, poor: { label: 'Poor', className: 'bg-red-500/20 text-red-400 border-red-500/30' } }`
   - Include a small dot indicator before the label text using a `span` with `size-1.5 rounded-full` and the matching color as `bg-*`
   - Add `data-testid="network-quality-badge"`

3. Create test files:
   - `src/hooks/useNetworkQuality.test.ts`: Mock `callRef` with a fake `peerConnection.getStats()` that returns fabricated `RTCStatsReport` (a Map with candidate-pair and outbound-rtp entries). Use `vi.useFakeTimers()` to advance the 2s interval. Test each quality tier by setting appropriate RTT and nack/packetsSent values.
   - `src/components/NetworkQualityBadge.test.tsx`: Render with each quality value, verify correct label text and that `data-testid` is present. Verify null quality renders nothing.
