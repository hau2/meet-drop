1. **Update `src/store/index.ts`:** Add `isScreenSharing: boolean` field (default false), `setScreenSharing: (sharing: boolean) => void` setter. Add to reset().

2. **Update `src/pages/RoomPage.tsx`:**
   - Destructure `callRef` from `useCall` (now exposed per Plan 02)
   - Add `const { isScreenSharing, startScreenShare, stopScreenShare } = useScreenShare(callRef, streamRef)`
   - Add `const networkQuality = useNetworkQuality(callRef, connectionState === 'connected')`
   - Pass new props to CallView: `isScreenSharing`, `onToggleScreenShare` (calls start or stop based on isScreenSharing), `networkQuality`, `callRef` (needed for fullscreen container ref is local to CallView, but callRef is not needed — fullscreen is self-contained in CallView)
   - Only pass `networkQuality` — CallView handles fullscreen internally

3. **Update `src/components/CallView.tsx`:**
   - **New props:** Add to `CallViewProps`: `isScreenSharing: boolean`, `onToggleScreenShare: () => void`, `networkQuality: NetworkQuality`
   - **Fullscreen (AV-07):**
     a. Add `containerRef = useRef<HTMLDivElement>(null)` on the outermost div (the one with `relative w-full h-screen bg-black`)
     b. Add `isFullscreen` state via `useState(false)`
     c. Add `useEffect` listening to `fullscreenchange` event on `document`: set `isFullscreen` to `!!document.fullscreenElement`
     d. Add `toggleFullscreen` function: if `!document.fullscreenElement` call `containerRef.current?.requestFullscreen()`, else call `document.exitFullscreen()`. Guard with `if (!document.fullscreenEnabled) return` for iOS Safari (open question from RESEARCH.md).
     e. Add fullscreen button in controls bar: import `Maximize2`, `Minimize2` from lucide-react. Render `Button` with `variant="secondary"` `size="lg"` `className="rounded-full"` `aria-label="Toggle fullscreen"`. Show `Maximize2` when not fullscreen, `Minimize2` when fullscreen.
   - **Screen share button:**
     a. Import `Monitor`, `MonitorOff` from lucide-react
     b. Add screen share toggle button in controls bar (between MediaControls and chat button): `Button` with `variant={isScreenSharing ? 'default' : 'secondary'}` `size="lg"` `className="rounded-full"` `aria-label={isScreenSharing ? 'Stop sharing' : 'Share screen'}`
     c. onClick calls `onToggleScreenShare`
     d. Show `MonitorOff` when sharing (to indicate "stop"), `Monitor` when not sharing
   - **Network quality badge:**
     a. Import `NetworkQualityBadge` from `./NetworkQualityBadge`
     b. Place badge in top-left corner of the call view: `absolute top-4 left-4 z-20`
     c. Render `<NetworkQualityBadge quality={networkQuality} />`
   - **Controls bar button order:** MediaControls | ScreenShare | Fullscreen | Chat | HangUp
   - Apply `ref={containerRef}` to the outermost div — this ensures fullscreen includes the controls overlay (Pitfall 4 from RESEARCH.md)

4. **Update `src/types/index.ts`:** Add `export type NetworkQuality = 'good' | 'fair' | 'poor' | null` — or import/re-export from useNetworkQuality hook. Prefer defining it in types/index.ts and importing in both the hook and badge for consistency with existing project pattern.
