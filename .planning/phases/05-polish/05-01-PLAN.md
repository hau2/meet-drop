<task type="auto" tdd="true">
  <name>Task 1: useTheme hook, ThemeToggle component, and theme initialization</name>
  <files>src/hooks/useTheme.ts, src/hooks/useTheme.test.ts, src/components/ThemeToggle.tsx, src/main.tsx, src/App.tsx</files>
  <behavior>
    - Test: useTheme returns isDark=true by default
    - Test: toggleTheme() removes 'dark' class from documentElement and sets sessionStorage 'theme' to 'light'
    - Test: toggleTheme() again adds 'dark' class back and sets sessionStorage 'theme' to 'dark'
    - Test: when sessionStorage has 'light', useTheme initializes isDark=false
  </behavior>
  <action>
1. In `src/main.tsx`, add synchronous theme init BEFORE `createRoot()` — read `sessionStorage.getItem('theme')`, if `'light'` remove `'dark'` class from `document.documentElement`, otherwise add `'dark'` class. This prevents flash of wrong theme (Pitfall 5 from RESEARCH.md).

2. Create `src/hooks/useTheme.ts`:
   - `useTheme()` returns `{ isDark: boolean, toggleTheme: () => void }`
   - On mount (`useLayoutEffect`), sync state with `document.documentElement.classList.contains('dark')`
   - `toggleTheme`: call `document.documentElement.classList.toggle('dark')`, update `sessionStorage.setItem('theme', isDark ? 'dark' : 'light')` (note: isDark is the NEW state after toggle), update local state
   - Do NOT use Zustand — theme is DOM-level, not app state, and sessionStorage is the persistence layer (PRIV-01 compatible)

3. Create `src/components/ThemeToggle.tsx`:
   - Import `Sun`, `Moon` from `lucide-react`, `Button` from `@/components/ui/button`, `useTheme` from hook
   - Render a `Button` with `variant="ghost"` `size="icon"` `aria-label="Toggle theme"`
   - Show `Moon` icon when dark, `Sun` icon when light
   - onClick calls `toggleTheme()`

4. In `src/App.tsx`, add `ThemeToggle` as a fixed button in top-right corner (`fixed top-4 right-4 z-50`) rendered outside the Router/Switch so it's visible on all pages.

5. Create `src/hooks/useTheme.test.ts` with the behaviors listed above. Mock `document.documentElement.classList` via jsdom (it works natively in jsdom). Mock `sessionStorage` via jsdom (natively available).
  </action>
  <verify>
    <automated>npx vitest run src/hooks/useTheme.test.ts --reporter=verbose</automated>
  </verify>
  <done>Dark theme is default, toggle switches theme and persists to sessionStorage, ThemeToggle button visible on all pages</done>
</task>

  <action>
1. Update `src/components/SelfViewOverlay.tsx`:
   - Add `useState<Corner>('br')` where `Corner = 'tl' | 'tr' | 'bl' | 'br'`
   - Define `CORNER_CLASSES` record mapping each Corner to Tailwind position classes: `{ br: 'bottom-4 right-4', bl: 'bottom-4 left-4', tr: 'top-20 right-4', tl: 'top-20 left-4' }` — use `top-20` instead of `top-4` to avoid overlapping the ThemeToggle button
   - Add `isDragging` state and `dragOffset` ref to track pointer offset from element origin
   - Add `style` for translate transform during drag (only while dragging, removed on snap)
   - `onPointerDown`: call `e.currentTarget.setPointerCapture(e.pointerId)`, record offset between pointer position and element bounding rect, set isDragging=true
   - `onPointerMove`: if dragging, compute translate position relative to the PARENT container (use `parentElement.getBoundingClientRect()`), set inline transform style
   - `onPointerUp`: determine which corner the overlay center is closest to (compare center X/Y to parent width/height midpoints), set corner state, clear transform style, set isDragging=false
   - Apply corner classes dynamically: `absolute ${CORNER_CLASSES[corner]} ...existing classes`
   - Add `touch-action: none` style and `cursor-grab` / `cursor-grabbing` classes
   - Keep existing `w-32 md:w-40 aspect-video rounded-lg overflow-hidden shadow-lg ring-1 ring-white/10 z-10` classes
